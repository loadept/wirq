# wirq — AGENTS.md

## What this is

A desktop MITM HTTP proxy built with [Wails v2](https://wails.io) — Go backend + Preact frontend, bundled as a native app.

## Project structure

```
main.go                 — Wails app entrypoint, flags (--version), binds App struct
app.go                  — App struct, holds *proxy.Manager, all frontend bindings
internal/config/        — JSON config read/write at ~/<AppName>/config.json
internal/proxy/         — Manager (log store), model types, CONNECT + plain HTTP handlers
internal/server/        — HTTP server lifecycle (start/stop)
frontend/src/           — Preact + Tailwind CSS v4 + Biome + Zod
frontend/wailsjs/       — Auto-generated Go→TS bindings (regenerated by wails dev/generate module)
```

## Dev commands

| Action | Command |
|---|---|
| Dev mode (live reload) | `wails dev` (Linux: `wails dev -tags webkit2_41`) |
| Build production | `wails build` (Linux: `wails build -tags webkit2_41`) |
| Go tests | `gotestsum --format testname -- -race ./internal/...` |
| Frontend lint/format | `bun run --bun biome check --write frontend/src/` (run from repo root or `frontend/`) |
| Frontend dev standalone | `cd frontend && bun run dev` (Vite, no backend) |
| Add frontend dependency | `cd frontend && bun add <pkg>` |

## Key conventions

- **Frontend package manager**: `bun` (not npm/pnpm/yarn). `bun install --frozen-lockfile` for prod, `bun install` for dev.
- **Config path**: OS user config dir + `/wirq/config.json`. Defaults: `127.0.0.1:3100`, dark theme.
- **Version**: set via `-ldflags` at build time; `main.go`'s `Version` var defaults to `"dev"`. Run `wails dev` or `./build/bin/wirq --version` to check.
- **Wails bindings** in `frontend/wailsjs/` are auto-generated. After changing Go structs or bind methods, run `wails generate module` or just `wails dev`.

## Backend architecture

- `proxy.Manager` holds logs in memory: `counter atomic.Int64`, `logs []*LogEntry`, `logMu sync.RWMutex`, `certMu sync.RWMutex`.
- Log store is bounded at **1000 entries** (`maxLogs`); older entries are evicted automatically.
- `LogSummary` (no bodies) emitted via `runtime.EventsEmit("proxy:log", summary)`.
- `LogEntry` (full detail with bodies) fetched via `GetLogDetail(id)` from the in-memory store.
- `ExportLogs(ids, filename)` — opens save dialog, reads from store, writes JSON.
- `ClearLogs()` — sets `p.logs = nil` under lock (counter NOT reset).
- `proxy:error` event emitted to frontend for runtime errors.
- `config.Manager.Write` uses atomic temp-file + rename to avoid corruption.

## Frontend architecture

### State management
- No external state library. Custom hooks + React context for toasts.
- `useBootstrap` — loads config, manages theme/settings modal lifecycle, runs `WindowShow` on mount.
- `useServer` — server lifecycle (start/stop), exposes `connected`, `starting`, `stopping`.
- `useLogList` — listens to `proxy:log` events, accumulates `LogSummary[]` (also bounded at 1000).
- `useLogDetail` — fetches full `LogEntry` by ID with cancel guard on unmount.
- `ToastProvider` — context-based toast system with auto-dismiss, mounted on `document.body` via portal.

### Form patterns
- Settings form uses **controlled** inputs with centralized `setValue(field, value)`:
  - updates `config` state
  - clears `issues` for that field
  - sets `dirty` flag
- `handleSubmit` reads from `config` state (not FormData), validates with Zod `SettingsSchema`.
- Validation errors stored as `Record<string, string>` in `issues` state.
- CSS uses `data-[error=true]:border-destructive` selector on inputs for visual feedback.
- Browse cert buttons use `handleBrowse(field)` which wraps `setValue`.
- Theme toggle is a controlled checkbox with live `onToggleTheme()` + state update.

### Error pattern
| Context | Communication |
|---|---|
| `useEffect` (async mount) | Internal `error` state → component `useEffect(error)` → toast |
| Action (click / submit) | `throw` from hook → component `try/catch` → toast |

### Filter system
- `matchFilter(log, text)` — space-separated tokens, supports `key:value`, `key:=exact`, `key:/regex/`.
- `parseTokens(text)` — parses filter text into `FilterToken[]` for chip display.
- `FilterModal` uses `<form onSubmit>`, shows parsed tokens as chips above buttons.
- Effect in `RequestList` clears selection only when the selected log no longer matches the filter.

### Custom hooks structure
```
frontend/src/lib/
  hooks/
    index.ts          — re-exports all hooks
    bootstrap.ts      — useBootstrap (config, theme, settings modal)
    server.ts         — useServer (start/stop proxy)
    logDetail.ts      — useLogDetail(logId)
    logList.ts        — useLogList(connected)
  providers/
    toast.ts          — ToastProvider + useToast context
  utils/
    filter.ts         — matchFilter, parseTokens, FilterToken
    http-format.ts    — requestToRaw, responseToRaw
```

### Types (local, not auto-generated)
- `types/index.ts`: re-exports from `logs.ts` and `toast.ts`, defines `Theme`, `ViewMode`, `PanelTab`.
- `types/logs.ts`: `LogSummary`.
- `types/toast.ts`: `Toast`, `ToastType`, `ToastContextValue`.
- Go types like `proxy.RequestLog`, `proxy.ResponseLog`, `proxy.LogEntry`, `config.ConfigDTO` come from `@wailsapp/models` (auto-generated).

## Frontend quirks

- Path aliases in Vite/TS: `@wailsapp/app`, `@wailsapp/runtime`, `@wailsapp/models` resolve into `frontend/wailsjs/`.
- Biome config excludes `style.css`, `wailsjs/`, `public/`, `dist/`, `build/`, `node_modules/`. Does NOT exclude `bun.lock`.
- Biome enforces `useBlockStatements: "error"` — all `if`/`else`/`for`/`while` must use curly braces.
- Style is all Tailwind utility classes + CSS `@import "tailwindcss"` (v4, no config file).
- Map over JSX children use explicit `key` props to avoid warnings.
- Dependency additions: `zod` (v4) for schema validation in `schemas/settings.ts`, `tailwind-merge` for modal class merging.
- All components use JSX syntax (`<Component />`), never function-call syntax.
- No dead code tolerated. Before closing a session, review every file for unused imports, dead variables, stale conditionals, etc.
- Modal closing: only via Escape key or explicit close button. No backdrop click — user hates accidental closes.
- Package manager: `bun` (not npm/npx/pnpm/yarn); use `bunx` for running binaries.
- Config on-disk format uses nested objects (`server.host`, `certAuthority.certPath`) but `ConfigDTO` is flat — `config.Manager` handles the translation.

### Typography

- **Font family**: `geist-mono` (monospace, variable axis 100–900). Applied on `:root`. Entire app is monospaced.
- **Declared but unused**: `firacode` (static 400, in CSS but not referenced by any component).
- **Base size**: `17px` on `:root` (`font-size: 17px`).
- **Font weights**: Only `400` (normal). No `font-bold`/`font-semibold` anywhere in the app.
- **Two text sizes, strict convention**:
  - `text-xs` (12px) — **UI chrome**: labels, titles, tab buttons, status text, buttons, toasts, errors, settings labels, filter chips, pretty-view labels, body-viewer banners, modal titles, confirm dialogs
  - `text-sm` (14px) — **data content**: request row path, request/response bodies, raw views, pretty-view values, settings inputs, filter input
- **Letter spacing**: `tracking-wider` (0.05em) on all labels, section headers, modal titles, tab buttons, pretty-view label columns, settings form labels.
- **No decorative typography**: no italic, no bold. Labels and section headers may use `uppercase` for visual hierarchy.

### Icon system

- **Library**: `lucide-preact`
- **Two sizes, strict convention** (plus one exception):
  - `h-3.5 w-3.5` (14px) — icon **with text** beside it: Lock (request row), Search (filter modal), Eye/File variants/Download (body-viewer buttons), Loader/Square/Play (start/stop button)
  - `h-4 w-4` (16px) — **standalone icon** (no adjacent text): Settings (header), Search/Download/BrushCleaning (toolbar), FolderSearch (browse cert), Copy (detail panel)
  - `h-2.5 w-2.5` (10px) — **exception**: connection status dot only
- **Icon color**: inherited from parent text color, or explicit `text-muted-foreground` (Lock, Search-in-filter), `text-primary` (Search-when-filtered), `text-foreground` (toolbar buttons).
- **No decorative icons**: every icon has a functional purpose.

### Borders and radius

#### Radius

`--radius: 0`. All Tailwind radius utilities compute to 0 or near-0:
- `rounded` → 0px (no-op). Used on: buttons, inputs, modals, toasts, chips, pre blocks.
- `rounded-full` → true circle. Used **only** on dark mode toggle knob and track.
- `rounded-sm`/`rounded-md`/`rounded-lg` → 0px (computed negatives, clamped by browser).

The app is **square-cornered by design**.

#### Border patterns

| Pattern | Where |
|---|---|
| `border-b border-border` | Header bottom, toolbar bottom, request row bottom, tab bar bottom, settings footer top |
| `border border-border` | Modal card, inputs, pre/code blocks, header tables, action buttons, binary previews, embeds |
